# Next.js 稅務文件生成器開發計畫

## **1. 專案摘要**

本計畫旨在建立一個基於 Next.js 的 Web 應用程式。使用者可以透過一個多頁面的問卷介面輸入個人資訊，應用程式將根據這些資訊填寫美國稅務表格（如 Form 8843 和 Form 1040-NR），並最終生成一個可供使用者下載的 PDF 檔案。

## **2. 現狀分析**

目前專案目錄下僅包含三個 Markdown 文件：

- `F1_Student_Tax_Guide_Framework.md`：定義了 F-1 學生的報稅指南。
- `Tax_Questions_Checklist.md`：模擬了 Web 應用程式的問題清單和互動流程。
- `README.md`：專案的說明文件。

專案中尚不存在任何 Next.js 相關的程式碼或設定。

## **3. 提議的變更**

我將分步驟執行以下變更，以完成此專案的後端部分：

### **步驟一：初始化 Next.js 專案**

- **操作：** 我將使用 `npx create-next-app@latest` 指令在目前的目錄下初始化一個新的 Next.js 專案。
- **設定：**
    - 使用 TypeScript。
    - 使用 ESLint 進行程式碼檢查。
    - 使用 App Router 來組織路由。

### **步驟二：安裝必要套件**

- **操作：** 我將使用 npm 安裝 `pdf-lib` 套件，用於處理 PDF 的讀取、修改和寫入。

### **步驟三：開發後端 API**

- **操作：** 我將在 `app/api/generate-pdf/route.ts` 中建立一個 API 路由，用於接收前端傳來的表單資料。
- **功能：** 此 API 將是整個功能的核心，負責協調資料和 PDF 生成邏輯。

### **步驟四：實現 PDF 生成邏輯**

- **操作：** 在 API 路由中，我會實現以下邏輯：
    1.  **獲取稅表範本：** 我會從 IRS 官方網站下載可填寫的 Form 8843 和 Form 1040-NR 的 PDF 範本，並將它們儲存在專案中。
    2.  **讀取 PDF 範本：** 使用 `pdf-lib` 讀取對應的稅表範本。
    3.  **填寫表單欄位：** 將從前端接收到的使用者資料，使用 `pdf-lib` 的功能填寫到 PDF 的相應欄位中。
    4.  **回傳 PDF：** 將填寫完成的 PDF 以 `application/pdf` 的形式回傳給前端。

## **4. 假設與決策**

- **框架與路由：** 使用 Next.js 搭配 App Router。
- **PDF 函式庫：** 使用 `pdf-lib` 進行伺服器端的 PDF 操作。
- **稅表來源：** 假設可以從 IRS 網站獲取到官方且欄位格式固定的可填寫 PDF 範本。
- **前端：** 根據您的指示，本次開發將**不包含**任何前端介面的建立。後續將由您提供前端程式碼進行對接。

## **5. 驗證步驟**

1.  啟動 Next.js 開發伺服器。
2.  我將提供一段 Node.js 或瀏覽器 `fetch` 的 JavaScript 程式碼片段。
3.  您可以使用此程式碼片段，帶上模擬的表單資料，向 `/api/generate-pdf` 發送一個 POST 請求。
4.  確認請求成功，並能下載一個 PDF 檔案。
5.  打開下載的 PDF 檔案，仔細核對其中的欄位是否已正確填入模擬的資料。
